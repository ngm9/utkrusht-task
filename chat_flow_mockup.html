Behind the chat

Each question maps to an input one of the existing modules already takes. Nothing new under the hood — just a friendlier doorway.

generate_input_files
generate_input_files/
  • Q1 → --competency-name
  • Q2 → --proficiency
  • Q3 → --role
↓
scenario_generator
scenario_generator/
  • Q4 → focus areas injected into scenario prompt
  • Q5 → domain flavor for the scenario narrative
  • defaults --count 6
↓
task prompt selection
task_generation_prompts/{level}/
  • Picked automatically from Q1 + Q2
  • e.g. Intermediate/Java_spring_boot_prompt.py
↓
generate-tasks
multiagent.py
  • Takes the three JSONs above
  • Runs eval loop, creates repo + gist
  • Stores in Supabase
